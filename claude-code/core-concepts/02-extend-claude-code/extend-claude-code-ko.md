# Claude Code 확장하기

> CLAUDE.md, Skills, 서브에이전트, 훅, MCP, 플러그인을 언제 사용해야 하는지 이해하기.

Claude Code는 코드를 추론하는 모델과 파일 작업, 검색, 실행, 웹 접근을 위한 [내장 도구](/en/how-claude-code-works#tools)를 결합합니다. 내장 도구는 대부분의 코딩 작업을 처리합니다. 이 가이드는 확장 레이어를 다룹니다: Claude가 아는 것을 커스터마이즈하고, 외부 서비스에 연결하고, 워크플로를 자동화하기 위해 추가하는 기능들입니다.

<Note>
  핵심 에이전트 루프가 어떻게 작동하는지에 대해서는 [Claude Code 작동 방식](/en/how-claude-code-works)을 참조하세요.
</Note>

**Claude Code가 처음이신가요?** 프로젝트 규칙을 위해 [CLAUDE.md](/en/memory)부터 시작하세요. 필요에 따라 다른 확장 기능을 추가하세요.

## 개요

확장 기능은 에이전트 루프의 각기 다른 부분에 연결됩니다:

* **[CLAUDE.md](/en/memory)**는 Claude가 매 세션마다 볼 수 있는 영구 컨텍스트를 추가합니다
* **[Skills](/en/skills)**는 재사용 가능한 지식과 호출 가능한 워크플로를 추가합니다
* **[MCP](/en/mcp)**는 Claude를 외부 서비스 및 도구에 연결합니다
* **[서브에이전트](/en/sub-agents)**는 격리된 컨텍스트에서 자체 루프를 실행하고 요약을 반환합니다
* **[훅](/en/hooks)**은 루프 외부에서 결정론적 스크립트로 실행됩니다
* **[플러그인](/en/plugins)**과 **[마켓플레이스](/en/plugin-marketplaces)**는 이러한 기능들을 패키징하고 배포합니다

[Skills](/en/skills)는 가장 유연한 확장 기능입니다. 스킬은 지식, 워크플로, 또는 지침을 포함하는 마크다운 파일입니다. `/deploy`와 같은 슬래시 명령으로 스킬을 호출하거나, Claude가 관련성이 있을 때 자동으로 로드할 수 있습니다. 스킬은 현재 대화에서 실행되거나 서브에이전트를 통해 격리된 컨텍스트에서 실행될 수 있습니다.

## 목표에 맞는 기능 선택

기능은 Claude가 매 세션마다 보는 상시 컨텍스트부터, 사용자나 Claude가 호출할 수 있는 온디맨드 기능, 특정 이벤트에서 실행되는 백그라운드 자동화까지 다양합니다. 아래 표는 사용 가능한 기능과 각각이 적합한 상황을 보여줍니다.

| 기능 | 하는 일 | 사용 시점 | 예시 |
| ------------- | ---------------------------------------------------------- | ------------------------------------------------------ | -------------------------------------------------------------------------------- |
| **CLAUDE.md** | 매 대화마다 로드되는 영구 컨텍스트 | 프로젝트 규칙, "항상 X 하기" 규칙 | "npm이 아닌 pnpm을 사용하세요. 커밋 전에 테스트를 실행하세요." |
| **스킬** | Claude가 사용할 수 있는 지침, 지식, 워크플로 | 재사용 가능한 콘텐츠, 참조 문서, 반복 작업 | `/review`로 코드 리뷰 체크리스트 실행; 엔드포인트 패턴이 있는 API 문서 스킬 |
| **서브에이전트** | 요약된 결과를 반환하는 격리된 실행 컨텍스트 | 컨텍스트 격리, 병렬 작업, 전문 작업자 | 많은 파일을 읽지만 핵심 결과만 반환하는 리서치 작업 |
| **MCP** | 외부 서비스에 연결 | 외부 데이터 또는 액션 | 데이터베이스 쿼리, Slack 게시, 브라우저 제어 |
| **훅** | 이벤트에서 실행되는 결정론적 스크립트 | 예측 가능한 자동화, LLM 미관여 | 파일 편집 후마다 ESLint 실행 |

**[플러그인](/en/plugins)**은 패키징 레이어입니다. 플러그인은 스킬, 훅, 서브에이전트, MCP 서버를 하나의 설치 가능한 단위로 번들링합니다. 플러그인 스킬은 네임스페이스로 구분되므로(`/my-plugin:review`와 같이) 여러 플러그인이 공존할 수 있습니다. 여러 저장소에서 동일한 설정을 재사용하거나 **[마켓플레이스](/en/plugin-marketplaces)**를 통해 다른 사람에게 배포하려면 플러그인을 사용하세요.

### 유사한 기능 비교

일부 기능은 비슷해 보일 수 있습니다. 다음은 이를 구별하는 방법입니다.

<Tabs>
  <Tab title="스킬 vs 서브에이전트">
    스킬과 서브에이전트는 서로 다른 문제를 해결합니다:

    * **스킬**은 어떤 컨텍스트에든 로드할 수 있는 재사용 가능한 콘텐츠입니다
    * **서브에이전트**는 메인 대화와 별도로 실행되는 격리된 작업자입니다

    | 측면 | 스킬 | 서브에이전트 |
    | --------------- | ---------------------------------------------- | ---------------------------------------------------------------- |
    | **정의** | 재사용 가능한 지침, 지식 또는 워크플로 | 자체 컨텍스트를 가진 격리된 작업자 |
    | **핵심 이점** | 컨텍스트 간 콘텐츠 공유 | 컨텍스트 격리. 작업은 별도로 수행되고 요약만 반환됨 |
    | **적합한 용도** | 참조 자료, 호출 가능한 워크플로 | 많은 파일을 읽는 작업, 병렬 작업, 전문 작업자 |

    **스킬은 참조형이거나 액션형일 수 있습니다.** 참조 스킬은 세션 전반에 걸쳐 Claude가 사용하는 지식을 제공합니다(API 스타일 가이드 등). 액션 스킬은 Claude에게 특정 작업을 수행하도록 지시합니다(배포 워크플로를 실행하는 `/deploy` 등).

    **서브에이전트는** 컨텍스트 격리가 필요하거나 컨텍스트 윈도우가 가득 차고 있을 때 사용하세요. 서브에이전트는 수십 개의 파일을 읽거나 광범위한 검색을 실행할 수 있지만, 메인 대화에는 요약만 전달됩니다. 서브에이전트의 작업은 메인 컨텍스트를 소모하지 않으므로, 중간 작업이 보일 필요가 없을 때도 유용합니다. 커스텀 서브에이전트는 자체 지침을 가질 수 있으며 스킬을 사전 로드할 수 있습니다.

    **둘을 결합할 수 있습니다.** 서브에이전트는 특정 스킬을 사전 로드할 수 있습니다(`skills:` 필드). 스킬은 `context: fork`를 사용하여 격리된 컨텍스트에서 실행될 수 있습니다. 자세한 내용은 [Skills](/en/skills)를 참조하세요.
  </Tab>

  <Tab title="CLAUDE.md vs 스킬">
    둘 다 지침을 저장하지만, 로드 방식과 목적이 다릅니다.

    | 측면 | CLAUDE.md | 스킬 |
    | ------------------------- | ---------------------------- | --------------------------------------- |
    | **로드** | 매 세션, 자동으로 | 온디맨드 |
    | **파일 포함 가능** | 예, `@path` 임포트로 | 예, `@path` 임포트로 |
    | **워크플로 트리거 가능** | 아니오 | 예, `/<name>`으로 |
    | **적합한 용도** | "항상 X 하기" 규칙 | 참조 자료, 호출 가능한 워크플로 |

    **CLAUDE.md에 넣을 것:** Claude가 항상 알아야 하는 것들: 코딩 규칙, 빌드 명령어, 프로젝트 구조, "절대 X 하지 않기" 규칙.

    **스킬에 넣을 것:** Claude가 가끔 필요한 참조 자료(API 문서, 스타일 가이드)이거나 `/<name>`으로 트리거하는 워크플로(배포, 리뷰, 릴리스).

    **경험 법칙:** CLAUDE.md는 ~500줄 이하로 유지하세요. 늘어나고 있다면 참조 콘텐츠를 스킬로 이동하세요.
  </Tab>

  <Tab title="MCP vs 스킬">
    MCP는 Claude를 외부 서비스에 연결합니다. 스킬은 그러한 서비스를 효과적으로 사용하는 방법을 포함하여 Claude가 아는 것을 확장합니다.

    | 측면 | MCP | 스킬 |
    | -------------- | ---------------------------------------------------- | ------------------------------------------------------- |
    | **정의** | 외부 서비스에 연결하기 위한 프로토콜 | 지식, 워크플로, 참조 자료 |
    | **제공하는 것** | 도구 및 데이터 접근 | 지식, 워크플로, 참조 자료 |
    | **예시** | Slack 통합, 데이터베이스 쿼리, 브라우저 제어 | 코드 리뷰 체크리스트, 배포 워크플로, API 스타일 가이드 |

    이 둘은 서로 다른 문제를 해결하며 함께 잘 작동합니다:

    **MCP**는 Claude에게 외부 시스템과 상호작용하는 능력을 부여합니다. MCP 없이는 Claude가 데이터베이스를 쿼리하거나 Slack에 게시할 수 없습니다.

    **스킬**은 Claude에게 그러한 도구를 효과적으로 사용하는 방법에 대한 지식과 `/<name>`으로 트리거할 수 있는 워크플로를 제공합니다. 스킬에는 팀의 데이터베이스 스키마와 쿼리 패턴이 포함될 수 있고, 팀의 메시지 포맷팅 규칙이 포함된 `/post-to-slack` 워크플로가 될 수도 있습니다.

    예시: MCP 서버는 Claude를 데이터베이스에 연결합니다. 스킬은 Claude에게 데이터 모델, 일반적인 쿼리 패턴, 다양한 작업에 어떤 테이블을 사용해야 하는지 알려줍니다.
  </Tab>
</Tabs>

### 기능의 레이어링 이해

기능은 여러 수준에서 정의할 수 있습니다: 사용자 전체, 프로젝트별, 플러그인을 통해, 또는 관리 정책을 통해. CLAUDE.md 파일을 하위 디렉토리에 중첩하거나 모노레포의 특정 패키지에 스킬을 배치할 수도 있습니다. 동일한 기능이 여러 수준에 존재할 때 레이어링 방식은 다음과 같습니다:

* **CLAUDE.md 파일**은 누적됩니다: 모든 수준이 Claude의 컨텍스트에 동시에 콘텐츠를 제공합니다. 작업 디렉토리와 상위의 파일은 시작 시 로드되고, 하위 디렉토리의 파일은 해당 디렉토리에서 작업할 때 로드됩니다. 지침이 충돌하면 Claude는 판단에 따라 이를 조정하며, 보통 더 구체적인 지침이 우선합니다. [Claude의 메모리 조회 방식](/en/memory#how-claude-looks-up-memories)을 참조하세요.
* **스킬과 서브에이전트**는 이름으로 덮어씁니다: 동일한 이름이 여러 수준에 존재하면 우선순위에 따라 하나의 정의가 적용됩니다(스킬: 관리 > 사용자 > 프로젝트; 서브에이전트: 관리 > CLI 플래그 > 프로젝트 > 사용자 > 플러그인). 플러그인 스킬은 충돌을 방지하기 위해 [네임스페이스](/en/plugins#add-skills-to-your-plugin)로 구분됩니다. [스킬 디스커버리](/en/skills#where-skills-live)와 [서브에이전트 범위](/en/sub-agents#choose-the-subagent-scope)를 참조하세요.
* **MCP 서버**는 이름으로 덮어씁니다: 로컬 > 프로젝트 > 사용자. [MCP 범위](/en/mcp#scope-hierarchy-and-precedence)를 참조하세요.
* **훅**은 병합됩니다: 등록된 모든 훅은 소스에 관계없이 매칭되는 이벤트에 대해 실행됩니다. [훅](/en/hooks)을 참조하세요.

### 기능 조합하기

각 확장 기능은 서로 다른 문제를 해결합니다: CLAUDE.md는 상시 컨텍스트를, 스킬은 온디맨드 지식과 워크플로를, MCP는 외부 연결을, 서브에이전트는 격리를, 훅은 자동화를 처리합니다. 실제 설정은 워크플로에 따라 이들을 조합합니다.

예를 들어, 프로젝트 규칙에는 CLAUDE.md를, 배포 워크플로에는 스킬을, 데이터베이스 연결에는 MCP를, 편집 후 린팅에는 훅을 사용할 수 있습니다. 각 기능이 가장 잘하는 것을 처리합니다.

| 패턴 | 작동 방식 | 예시 |
| ---------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| **스킬 + MCP** | MCP가 연결을 제공하고, 스킬이 Claude에게 그것을 잘 사용하는 방법을 알려줌 | MCP가 데이터베이스에 연결하고, 스킬이 스키마와 쿼리 패턴을 문서화 |
| **스킬 + 서브에이전트** | 스킬이 병렬 작업을 위해 서브에이전트를 생성 | `/review` 스킬이 격리된 컨텍스트에서 작동하는 보안, 성능, 스타일 서브에이전트를 실행 |
| **CLAUDE.md + 스킬** | CLAUDE.md가 상시 규칙을 보유하고, 스킬이 온디맨드로 로드되는 참조 자료를 보유 | CLAUDE.md는 "API 규칙을 따르세요"라고 하고, 스킬에는 전체 API 스타일 가이드가 포함됨 |
| **훅 + MCP** | 훅이 MCP를 통해 외부 액션을 트리거 | 편집 후 훅이 Claude가 중요 파일을 수정했을 때 Slack 알림을 전송 |

## 컨텍스트 비용 이해

추가하는 모든 기능은 Claude의 컨텍스트를 일부 소모합니다. 너무 많으면 컨텍스트 윈도우가 가득 찰 수 있지만, 노이즈를 추가하여 Claude의 효율성을 떨어뜨릴 수도 있습니다; 스킬이 올바르게 트리거되지 않거나 Claude가 규칙을 놓칠 수 있습니다. 이러한 트레이드오프를 이해하면 효과적인 설정을 구축하는 데 도움이 됩니다.

### 기능별 컨텍스트 비용

각 기능은 서로 다른 로딩 전략과 컨텍스트 비용을 가집니다:

| 기능 | 로드 시점 | 로드 내용 | 컨텍스트 비용 |
| --------------- | ------------------------- | --------------------------------------------- | -------------------------------------------- |
| **CLAUDE.md** | 세션 시작 | 전체 내용 | 매 요청마다 |
| **스킬** | 세션 시작 + 사용 시 | 시작 시 설명, 사용 시 전체 내용 | 낮음 (매 요청마다 설명만)* |
| **MCP 서버** | 세션 시작 | 모든 도구 정의 및 스키마 | 매 요청마다 |
| **서브에이전트** | 생성 시 | 지정된 스킬이 포함된 새로운 컨텍스트 | 메인 세션과 격리됨 |
| **훅** | 트리거 시 | 없음 (외부에서 실행) | 제로, 훅이 추가 컨텍스트를 반환하지 않는 한 |

\*기본적으로 스킬 설명은 세션 시작 시 로드되어 Claude가 언제 사용할지 결정할 수 있습니다. 스킬의 프론트매터에서 `disable-model-invocation: true`를 설정하면 수동으로 호출할 때까지 Claude에게 완전히 숨겨집니다. 이렇게 하면 직접 트리거하는 스킬의 컨텍스트 비용이 제로로 줄어듭니다.

<Tabs>
  <Tab title="CLAUDE.md">
    **시점:** 세션 시작

    **로드되는 내용:** 모든 CLAUDE.md 파일의 전체 내용 (관리, 사용자, 프로젝트 수준).

    **상속:** Claude는 작업 디렉토리에서 루트까지 CLAUDE.md 파일을 읽고, 해당 파일에 접근할 때 하위 디렉토리의 파일을 발견합니다. 자세한 내용은 [Claude의 메모리 조회 방식](/en/memory#how-claude-looks-up-memories)을 참조하세요.

    <Tip>CLAUDE.md를 ~500줄 이하로 유지하세요. 참조 자료는 온디맨드로 로드되는 스킬로 이동하세요.</Tip>
  </Tab>

  <Tab title="스킬">
    스킬은 Claude의 도구 키트에 있는 추가 기능입니다. 참조 자료(API 스타일 가이드 등)이거나 `/<name>`으로 트리거하는 호출 가능한 워크플로(`/deploy` 등)일 수 있습니다. 일부는 내장되어 있으며, 직접 만들 수도 있습니다. Claude는 적절할 때 스킬을 사용하거나, 직접 호출할 수 있습니다.

    **시점:** 스킬의 구성에 따라 다릅니다. 기본적으로 설명은 세션 시작 시 로드되고 전체 내용은 사용 시 로드됩니다. 사용자 전용 스킬(`disable-model-invocation: true`)은 호출할 때까지 아무것도 로드되지 않습니다.

    **로드되는 내용:** 모델 호출 가능 스킬의 경우, Claude는 매 요청에서 이름과 설명을 봅니다. `/<name>`으로 스킬을 호출하거나 Claude가 자동으로 로드하면, 전체 내용이 대화에 로드됩니다.

    **Claude의 스킬 선택 방식:** Claude는 스킬 설명과 작업을 매칭하여 관련성을 판단합니다. 설명이 모호하거나 겹치면, Claude가 잘못된 스킬을 로드하거나 도움이 될 스킬을 놓칠 수 있습니다. Claude에게 특정 스킬을 사용하도록 하려면 `/<name>`으로 호출하세요. `disable-model-invocation: true`가 설정된 스킬은 호출할 때까지 Claude에게 보이지 않습니다.

    **컨텍스트 비용:** 사용 전까지 낮음. 사용자 전용 스킬은 호출 전까지 비용 제로.

    **서브에이전트에서:** 스킬은 서브에이전트에서 다르게 작동합니다. 온디맨드 로딩 대신, 서브에이전트에 전달된 스킬은 시작 시 컨텍스트에 완전히 사전 로드됩니다. 서브에이전트는 메인 세션의 스킬을 상속하지 않으므로 명시적으로 지정해야 합니다.

    <Tip>부수 효과가 있는 스킬에는 `disable-model-invocation: true`를 사용하세요. 이렇게 하면 컨텍스트가 절약되고 직접만 트리거할 수 있습니다.</Tip>
  </Tab>

  <Tab title="MCP 서버">
    **시점:** 세션 시작.

    **로드되는 내용:** 연결된 서버의 모든 도구 정의 및 JSON 스키마.

    **컨텍스트 비용:** [도구 검색](/en/mcp#scale-with-mcp-tool-search) (기본적으로 활성화)은 MCP 도구를 컨텍스트의 10%까지 로드하고 나머지는 필요할 때까지 지연합니다.

    **안정성 참고:** MCP 연결은 세션 중에 조용히 실패할 수 있습니다. 서버가 연결 해제되면 도구가 경고 없이 사라집니다. Claude가 더 이상 존재하지 않는 도구를 사용하려 할 수 있습니다. Claude가 이전에 접근할 수 있었던 MCP 도구를 사용하지 못하는 것을 발견하면 `/mcp`로 연결을 확인하세요.

    <Tip>`/mcp`를 실행하여 서버별 토큰 비용을 확인하세요. 활발히 사용하지 않는 서버는 연결을 해제하세요.</Tip>
  </Tab>

  <Tab title="서브에이전트">
    **시점:** 온디맨드, 사용자나 Claude가 작업을 위해 생성할 때.

    **로드되는 내용:** 다음을 포함하는 새로운 격리된 컨텍스트:

    * 시스템 프롬프트 (캐시 효율성을 위해 부모와 공유)
    * 에이전트의 `skills:` 필드에 나열된 스킬의 전체 내용
    * CLAUDE.md 및 git 상태 (부모로부터 상속)
    * 리드 에이전트가 프롬프트에서 전달하는 컨텍스트

    **컨텍스트 비용:** 메인 세션과 격리됨. 서브에이전트는 대화 기록이나 호출된 스킬을 상속하지 않습니다.

    <Tip>전체 대화 컨텍스트가 필요 없는 작업에 서브에이전트를 사용하세요. 격리 덕분에 메인 세션이 비대해지는 것을 방지합니다.</Tip>
  </Tab>

  <Tab title="훅">
    **시점:** 트리거 시. 훅은 도구 실행, 세션 경계, 프롬프트 제출, 권한 요청, 압축 등 특정 라이프사이클 이벤트에서 발생합니다. 전체 목록은 [훅](/en/hooks)을 참조하세요.

    **로드되는 내용:** 기본적으로 없음. 훅은 외부 스크립트로 실행됩니다.

    **컨텍스트 비용:** 제로, 훅이 대화에 메시지로 추가되는 출력을 반환하지 않는 한.

    <Tip>훅은 Claude의 컨텍스트에 영향을 줄 필요가 없는 부수 효과(린팅, 로깅)에 이상적입니다.</Tip>
  </Tab>
</Tabs>
